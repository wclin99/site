## 生产力

今天花时间将项目管理从 Linear 转移到用飞书多维表格来搭建了。Linear 此前也是用了一段时间后又搁置了（这也不是第一次情况），总结有以下几方面原因：

1. 客观：配置复杂，有许多用不上的参数例如`cycle`、工作量预估、分配人
2. 客观：自动化能力稍弱（难以做到满意的客制化自动化）
3. 客观：项目逻辑比较重，对于项目有 `on track`、里程碑等配置，比较复杂
4. 客观：生态比较孤立，无法与国内的 IM 打通数据、再扩展能力（例如消息提醒）
5. 主观：即便是用了一两个月，但习惯没有养成

这里临时想的这几个方面的原因还不够系统和抽象，后续要重新回顾一下，捋清晰些。

综合各方面原因，最终选择用飞书多维表格来重新搭建整个项目管理系统，至少从现在来看，客制化程度和灵活程度以及数据打通的能力是能够符合需求的，不过 UI 还是硬伤。
